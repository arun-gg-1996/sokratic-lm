## User requirements collected across this UI cycle

### Layout and structure
- Left rail and main chat should be clearly separated.
- Left and right sides should scroll independently.
- Sidebar should feel Claude-like (visual hierarchy, spacing, icon scale, hover feel).
- Account block should be pinned at bottom-left.
- Chat should remain clean with minimal chrome in non-debug mode.

### Debug behavior
- Debug-only data (trace cards, export chat, student profiles, internals) should appear **only** in debug mode.
- Clicking account user (Arun) should open a hover/popover menu.
- Account popover should include only:
  - user name,
  - Learning Insights entry,
  - Debug toggle,
  - logout.
- No “workspace menu” wording.

### Composer/chat input
- Composer should be visually large, rectangular with rounded corners, and pinned to bottom like Claude.
- `+` and send controls should be **inside** composer.
- Send control should be compact.
- Composer should be slightly darker gray.
- Do not show “multimodal later” helper text outside chat box.

### Visual design
- Light/white non-debug theme requested.
- Use Sokratic font style consistently.
- Remove default Streamlit banner overlap.
- Avoid emojis for labels/navigation.
- Bot icon should be larger and better aligned vertically with assistant bubbles.

## Likely technical root causes (for Claude analysis)

- Heavy reliance on brittle CSS selectors against Streamlit internals (including `:has(...)` + testid targeting).
- Mixed fixed-position + normal-flow blocks causing overlap/reflow conflicts.
- Competing render branches (debug vs non-debug) sharing session state and layout anchors.
- Streamlit rerun model interacting poorly with dynamic CSS/conditional containers.
- Custom composer built from standard widgets without isolated bottom-dock container contract.
