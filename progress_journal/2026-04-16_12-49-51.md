## What was implemented

### `ingestion/extract.py` — fully implemented

Reads the 1317-page OpenStax A&P 2e PDF and produces tagged raw elements
saved to `data/processed/raw_elements_ot.jsonl`.

#### Key decisions

**Font-based heading detection (PyMuPDF `get_text("dict")`)**
- Section headings (e.g. "11.1 Interactions of Skeletal Muscles"): Mulish-SemiBold ≥ 12.5pt
- Subsection headings (e.g. "The Lever System of Muscle and Bone Interactions"): Mulish-SemiBold 10.0–12.4pt
- Body text: IBMPlexSans-Regular 8.5–10.0pt
- Figure captions: any font ≤ 8.0pt, starting with "FIGURE X.X"
- Page headers/footers filtered by y-position: y < 45 or y > 740 (page height = 792)

**Lazy section tracking (important correctness decision)**

The TOC says "section 11.2 starts at page 416" but the TOP of page 416 still
has section 11.1 content (before the inline "11.2 Naming Skeletal Muscles"
heading appears mid-page). If we naively apply the TOC section at the page
level, those first paragraphs get mislabeled as 11.2.

Fix: sections update ONLY when an inline section heading is detected by the
font classifier. The TOC is used only for chapter-level transitions (chapters
do start cleanly at the top of their page). Running state carries section_num
and section_title forward across pages until an inline heading resets them.

**Learning-objectives bullet filtering**

"LEARNING OBJECTIVES" and "CHAPTER OBJECTIVES" blocks appear at the start of
each section with bullet lists ("• Describe the criteria..."). These are NOT
content — they're curriculum metadata. The body font (IBMPlexSans-Regular) is
shared with content, so a `saw_lo_header` flag is set when a block containing
"LEARNING OBJECTIVES", "CHAPTER OBJECTIVES", "By the end of this section", or
"After studying this chapter" is encountered. Bullet blocks immediately
following (starting with "•") are skipped until a non-bullet body block resets
the flag. Content bullets in the body (e.g. "• Anabolism is the process...
") are preserved.

**Table extraction (pdfplumber) + overlap masking**

Tables extracted via pdfplumber `find_tables()`. Table bounding boxes (which
use the same top-left coordinate system as PyMuPDF) are passed into
`_extract_page_text_elements()` to mask out text blocks that overlap table
regions, preventing double-counting.

Raw table rows are pipe-delimited and later converted to prose via Claude
(`claude-sonnet-4-5`, `table_to_text` prompt from `config.yaml`). One
Anthropic client is created once in `extract_pdf()` and reused for all
conversions (not re-created per table). Rate-limit retries with 15s backoff.

**Blocks sorted by y0 before processing**

PyMuPDF returns blocks in PDF content-stream order, which is not always
top-to-bottom. Blocks are sorted by y0 before processing so heading
detection and caption buffering work in reading order.

## Issues encountered and fixes

**Issue 1: First paragraph on section-boundary pages mislabeled**
- Symptom: page 416 (where section 11.2 starts per TOC) had its opening
  paragraph tagged as "11.2" even though it was finishing 11.1 content.
- Root cause: the main loop was applying the TOC section_num to section_meta
  at the start of every page where the TOC entry changed.
- Fix: removed TOC-based section updates from the main loop entirely. Sections
  now update only via inline heading detection inside
  `_extract_page_text_elements`. Only chapter transitions still use the TOC.

**Issue 2: Learning-objectives bullet lists extracted as paragraphs**
- Symptom: "• Describe the criteria used to name skeletal muscles • Explain
  how..." appeared as paragraph elements.
- Root cause: the bullet text uses IBMPlexSans-Regular at 9pt (same font as
  body), SKIP_PREFIXES only matched the preceding "By the end of this
  section" block, not the actual bullet block.
- Fix: added `saw_lo_header` state flag; set True on LO trigger phrases,
  reset False on first non-bullet body block after the LO section.

**Issue 3: One garbage table from pdfplumber**
- One table conversion returned "I'd be happy to help but the table appears
  to be just a column of numbers without context" — pdfplumber picked up a
  numeric cross-reference structure (likely a reference index) as a table.
- Impact: minor — one low-quality element. The chunk.py min-length filter
  (100 chars) or proposition step will handle it.
- No fix applied (edge case, not worth adding special logic).
