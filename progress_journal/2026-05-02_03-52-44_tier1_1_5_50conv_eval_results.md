## Issues surfaced

### Real bugs (worth fixing)

1. **Wrong-topic locks**: 6/51 (11.8%). The system locked the WRONG
   chapter on certain S4/S5 abstract phrasings. E.g. S5 expected
   "Skeletal System" but locked "ventral horn" (spinal cord). The
   topic-engagement classifier struggles on student-driven
   paraphrases.

2. **Long-tail retry loops**: ~6/59 convos took 28-50+ minutes wall
   each. Same pattern as part 1's S2 outliers. The S6 long-tails
   pushed the part 2 run from ~50 min projected to ~7 hr. They DID
   complete with reached=True, so it's "slow not broken" — likely
   Dean QC reject-and-retry on certain prompt/anchor combos. Each
   long-tail is 27-54 API calls per turn (vs ~10 normal).

3. **Empty-subsection chunks silently filtered from semantic_top
   vote**: 1,012 of 7,516 chunks (13.5%) have empty subsection_title
   (because OpenStax sections like "Nervous Tissue Mediates
   Perception" don't subdivide). The dean's vote algorithm at
   `dean.py:~1340` does `if not key[2]: continue` — skips these.
   Surfaced live during a manual UI test where "what's the gap
   between the neurons" hit the synapse-defining chunk at score 1.0
   but the system fell to fuzzy-tier=borderline → cards.

### Non-bugs (false positives in offline grep detector)

1. "31% pre-reveal locked-answer leak" → mostly closure messages and
   topic-lock acks; real rate <5%.
2. "46% strong-affirm sycophancy" → false positives on legitimate
   earned affirmations (S1 100% reach with 67% "sycophancy" hit rate
   is the symptom). The runtime Haiku classifier is state-aware and
   doesn't fire on these.
3. "Hint leakage 19.6%" → 8 of 13 hits matched on `"sounds like"`
   in empathic phrases ("It sounds like you're stuck"); not actual
   rhyme leaks. Real hint-leak rate ~0.

### Conversation-quality observations (from sampled transcripts)

- All 6 profile transcripts use `What you got right / What to correct
  next` structured feedback — clean Socratic shape.
- Closures are graceful even on failure modes (S5 close: *"This
  session covered the epidermal layers… The correct order is…"*).
- Profile-aware behavior is working: tutor pulls S4's wandering
  attention back, lets S3 grind to slow win, validates S6's hedged
  correct pieces.

## Next (post-eval)

Two priority paths visible from the eval:

1. **Section-level lock fix** (~30 min) — flip the `if not key[2]:
   continue` in dean.py's `semantic_top` vote so chunks with empty
   subsection can lock at section level. Unblocks the synapse case
   (and 8 other section-only TOC entries).
2. **LLM topic-classifier** (~3 hr) — replaces fuzzy+semantic_top with
   a Haiku call that reads (query, list_of_TOC_entries) and returns
   the best match. Solves the entire class of brittle-fuzzy-matching
   bugs at once. Same architectural pattern as the Haiku classifiers
   we already shipped for hint-leak / sycophancy / off-domain.

Plus the long-tail bug investigation (Dean retry-cap or whatever's
driving 27-54 calls/turn on certain S2/S6 topics).

These are the next round; the current branch is in a stable, paper-
ready state for the eval results.
