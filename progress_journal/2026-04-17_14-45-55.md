## What we changed

### 1) Dean/Teacher prompt and control flow tightening
- Added Dean preflight guidance into Teacher Socratic drafting, so Teacher sees QC constraints before drafting.
- Kept Dean as the supervisor for QC and rewrite path (single-pass revised draft apply when available).
- Reduced repetitive/generic tutor lead-ins and enforced one-question Socratic structure more consistently.

### 2) Latency reductions
- Removed old Dean tool-loop style setup pass behavior.
- Moved retrieval + answer lock to Python fast-path in Dean (`dean.python_retrieval`, `dean.python_lock_answer`).
- Setup classification is now a single Dean LLM call with smaller context window.
- Testing model stack switched to Haiku for speed in this phase.

### 3) QC parse reliability hardening
- Strengthened JSON extraction/parsing in Dean QC path:
  - fenced JSON handling
  - smart quote normalization
  - trailing comma cleanup
  - balanced-brace extraction
  - Python-literal fallback for malformed dict-like outputs
- QC calls now run with `temperature=0` and stricter output constraints.

### 4) Confidence logic redesign (important)
- Initial attempt used a downgrade gate (`correct -> partial_correct`) for hedged answers.
- Revised per review: **removed downgrade behavior** and moved to cleaner design:
  - Keep categorical `student_state` untouched.
  - Add numeric `confidence_score` per answer.
  - Add running `student_mastery_confidence` across turns.
  - Gate `student_reached_answer` using `(student_state == correct) AND confidence >= threshold`.
- Added config threshold: `thresholds.reached_answer_confidence = 0.72`.

### 5) Trace and UI observability improvements
- Cache visibility now always explicit (including zeros):
  - per-turn: `read:X write:Y`
  - per-call badges: `💾hit:X`, `✍write:Y`
- Added explicit Dean confidence trace entries:
  - `dean.confidence_score` with answer/mastery confidence and reached decision.
- Internal state panel now shows:
  - latest answer confidence
  - running mastery confidence
  - confidence sample count

### 6) Memory persistence behavior without Qdrant
- Since Qdrant is not attached, memory flush now reports explicit status instead of generic failure:
  - `skipped_qdrant_unavailable`
- Memory remains non-fatal; sessions continue normally.
