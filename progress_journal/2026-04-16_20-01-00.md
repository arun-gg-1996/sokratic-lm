## What was decided

### LeakGuard simplified
- Dropped Level 1 (exact match — answer always hidden in Teacher's question, not verbatim)
- Dropped Level 2 (cosine — redundant since Level 3 catches everything)
- **Final: Level 3 only** — Dean LLM entailment check embedded in quality check call. No extra API cost.

### check_student_answer
- Dropped cosine similarity approach (inaccurate for paraphrases)
- Dean LLM reasons about correctness directly — it already has locked_answer + student message in context. More accurate, zero extra API call.

### student_state labels — 6 total
`correct` | `partial_correct` | `incorrect` | `question` | `irrelevant` | `low_effort`
- Added `partial_correct`: student gets part right — Teacher affirms + probes gap (different from incorrect)
- Hint advance: ONLY on `incorrect` (genuine wrong attempt). `question` and `irrelevant` keep hint level.
- `low_effort`: counter++, hint stays. After 3 consecutive → advance anyway, reset counter.

### Turn limit — one number does two jobs
- `max_turns: 25` in config.yaml
- Summarizer fires at `turn_count >= 25 - summarizer_keep_recent` (turn 15+)
- `after_dean` forces assessment at `turn_count >= max_turns`
- No separate `max_session_turns` config

### Dean structured output
- Option B: `submit_turn_evaluation` Anthropic tool_use
- Dean always returns valid JSON schema — no free-text parsing risk
- Fields: student_state, student_reached_answer, hint_level, locked_answer, search_needed, critique

### Prompt wrappers — 7 total, graph-driven
- LLM never decides routing. Graph state machine determines which wrapper fires.
- dean: `_setup_call`, `_quality_check_call`, `_assessment_call`, `_memory_summary_call`
- teacher: `draft_rapport`, `draft_socratic`, `draft_clinical`
- Rapport: procedural (memory_manager.load() + teacher.draft_rapport()) — Dean not involved

### Dean quality check = EULER criteria (real-time)
- question_present, relevance, helpful, no_reveal — same 4 criteria
- Dean checks live every turn. EULER script validates offline on saved logs. Same rubric, different timing.

### Retry policy
- One retry only. Teacher fails twice → Dean writes generic safe fallback ("That's interesting — what else do you recall about this region?")

### Debug tracking
- `debug: dict` field in TutorState: api_calls, input_tokens, output_tokens, interventions
- Updated after every Anthropic call. Shown in Streamlit debug panel.
- LangSmith (free tier) for deep cross-session trace inspection.

### Graph routing safety
- `after_dean` updated: checks `student_reached_answer` OR `hint_level > max_hints` OR `turn_count >= max_turns`
- No internal loops — each graph.invoke() ends at END. Streamlit calls invoke() per student message.
- Dean output validated as JSON before writing to state. Missing fields → use current state values.

### guiding_question_chain — DROPPED
- Was Dean scripting Teacher's questions. Violates responsibility delineation.
- Teacher now forms questions freely based on hint_level + retrieved_chunks. hint_level provides specificity guidance (1=broad, 2=narrower, 3=direct push).

### check_student_answer tool — REMOVED from ALL_TOOLS
- Dean reasons about correctness in its setup call (already has context). No separate tool.

### embed_fn removed from DeanAgent
- Was needed for cosine checks. Dropped with LeakGuard L1/L2. DeanAgent no longer needs embed_fn.
