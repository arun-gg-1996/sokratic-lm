# Phase 1 Gate Refix (Cache + Anchor/Loop Stabilization)

Date: 2026-04-20

## Why this patch was needed
- Phase 1 gate failed (`cache_read=0`, `cache_write=0`).
- `teacher_base` / `dean_base` were effectively stubs, and cached blocks were too small for Anthropic cache writes.
- Anchor lock failure caused repeated "narrower focus" loops and re-retrieval.
- Retrieval call-count occasionally reached `2` in one session.

## Changes made

### 1) Cache block refix (teacher + dean)
- Updated:
  - `conversation/teacher.py::_cached_system`
  - `conversation/dean.py::_cached_system`
- New behavior:
  - Build `cached_primary = role_base + wrapper_delta + chunks + history`.
  - If `cached_primary` is still below cache minimum (1024 estimated tokens), temporarily promote `turn_deltas` into the cached block.
  - This prevents all-zero cache metrics when stable blocks are too short.

### 2) Retrieval single-fire guard + no chunk wipe on relock loops
- Updated:
  - `conversation/dean.py::run_turn` topic-selection branch
  - `conversation/dean.py::_retrieve_on_topic_lock`
- New behavior:
  - Retrieval fires at most once per session (`debug.retrieval_calls <= 1`).
  - If retrieval already fired, subsequent topic re-selections do not clear chunks and do not re-query.
  - Anchor-fail return path now keeps `retrieved_chunks` instead of wiping them.

### 3) Anchor lock robustness
- Updated:
  - `conversation/dean.py::_lock_anchors_call`
- New behavior:
  - Increased lock-anchor output budget (`max_tokens: 360`) to reduce JSON truncation.
  - If JSON parse fails, one fallback answer extraction path is used (`_extract_answer_parametric` + sanitize), rather than hard-empty anchors only.

### 4) Duplicate-call guard for expensive assessment wrappers
- Updated:
  - `conversation/dean.py::_clinical_turn_call`
  - `conversation/dean.py::_close_session_call`
- New behavior:
  - Added request fingerprint-based dedupe guard.
  - If identical input payload is invoked twice, second call reuses prior parsed result and logs dedupe event in trace.

### 5) Prompt/base consistency cleanup
- Updated:
  - `config/base.yaml`
  - `config.yaml`
- Changes:
  - Replaced minimal `teacher_base`/`dean_base` stubs with substantive shared scaffolding.
  - Standardized lock-answer rule to concise phrase (`typically 2-10`, hard max `15` words).
  - Added brief-rationale constraint (`<=25 words`) for anchor lock output.
